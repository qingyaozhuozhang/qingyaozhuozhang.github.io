## 1. 项目概览

### 1.1 项目定位

BehaviorTree.ROS2 是一个将 [BehaviorTree.CPP](https://github.com/BehaviorTree/BehaviorTree.CPP) 与 ROS2 做深度封装的库。它提供了标准的方式来实现：

- **Behavior Tree Executor**：通过 ROS Action 接口执行行为树
- **Action 客户端节点**：在 BT 中异步调用 ROS Action Server
- **Service 客户端节点**：在 BT 中异步调用 ROS Service
- **Topic 订阅节点**：在 BT 中接收 Topic 消息
- **Topic 发布节点**：在 BT 中发布 Topic 消息

核心目标：

1. **最小化样板代码** — 用户只需实现关键回调
2. **异步非阻塞** — ROS 通信不会阻塞 BT 主循环的 `tick()`

### 2.3 TreeExecutionServer 详解

[`TreeExecutionServer`](behaviortree_ros2/include/behaviortree_ros2/tree_execution_server.hpp) 是整个系统的入口，它：

1. **托管 ROS Action Server**：接收 `ExecuteTree` 类型的 Goal
2. **管理 BehaviorTreeFactory**：注册插件和 XML 树
3. **执行行为树**：在独立线程中按频率 tick
4. **提供扩展钩子**：允许用户通过派生来定制行为

可覆写的虚函数：

| 虚函数                                            | 调用时机                 | 用途                         |
| ------------------------------------------------- | ------------------------ | ---------------------------- |
| `onGoalReceived(tree_name, payload)`              | 收到 Goal 时，创建树之前 | 验证或拒绝请求               |
| `onTreeCreated(tree)`                             | 树创建之后               | 初始化日志器/黑板            |
| `registerNodesIntoFactory(factory)`               | 注册阶段                 | 手动注册额外节点             |
| `onLoopAfterTick(status)`                         | 每次 tick 之后           | 中断执行（返回有效值则停止） |
| `onLoopFeedback()`                                | 每次 tick 之后           | 发送反馈消息给 Action Client |
| `onTreeExecutionCompleted(status, was_cancelled)` | 树执行结束               | 自定义返回消息               |

### 2.4 ROS 通信接口定义

#### ExecuteTree.action

```
# 文件: btcpp_ros2_interfaces/action/ExecuteTree.action

#### Request ####
string target_tree    # 要执行的树名（对应 XML 中 BehaviorTree ID）
string payload        # 可选的自定义负载数据
---
#### Result ####
NodeStatus node_status    # 树的最终状态
string return_message     # 返回消息或错误信息
---
#### Feedback ####
string message            # 执行过程中的反馈消息
```

#### NodeStatus.msg

```
uint8 IDLE = 0
uint8 RUNNING = 1
uint8 SUCCESS = 2
uint8 FAILURE = 3
uint8 SKIPPED = 4

uint8 status
```

## 3. XML 决策树导入 ROS2 的完整流程

### 3.1 配置文件定义加载路径

YAML 配置文件（如 [`sample_bt_executor.yaml`](btcpp_ros2_samples/config/sample_bt_executor.yaml)）定义了要加载的插件和行为树：

```yaml
bt_action_server:
  ros__parameters:
    action_name: "behavior_server"  # Action Server 名称（默认 bt_action_server）
    tick_frequency: 100             # tick 频率 Hz（默认 100）
    groot2_port: 1667               # Groot2 调试端口（默认 1667）
    ros_plugins_timeout: 1000       # ROS 插件超时 ms（默认 1000）

    plugins:                        # 插件目录列表（格式: package_name/subfolder）
      - behaviortree_cpp/bt_plugins
      - btcpp_ros2_samples/bt_plugins

    behavior_trees:                 # XML 树目录列表（格式: package_name/subfolder）
      - btcpp_ros2_samples/behavior_trees
```

**路径解析规则**：`package_name/subfolder` 会被 [`GetDirectoryPath()`](behaviortree_ros2/src/bt_utils.cpp:50) 解析为：

```
ament_index_cpp::get_package_share_directory("package_name") + "/subfolder"
```

即实际路径为 ROS2 包安装目录下的 `share/package_name/subfolder`。

参数完整说明见 [`bt_executor_parameters.md`](behaviortree_ros2/bt_executor_parameters.md)。

### 3.2 启动时自动注册

`TreeExecutionServer` 构造时，通过一个单次触发的 wall timer 异步调用 [`executeRegistration()`](behaviortree_ros2/src/tree_execution_server.cpp:110)：

```cpp
// tree_execution_server.cpp 构造函数中
p_->single_shot_timer = node_->create_wall_timer(
    std::chrono::milliseconds(1), [this]() {
      if(!p_->factory_initialized_) {
        executeRegistration();
      }
      p_->single_shot_timer->cancel();  // 只执行一次
    });
```

`executeRegistration()` 的执行顺序：

```cpp
void TreeExecutionServer::executeRegistration()
{
  p_->factory.clearRegisteredBehaviorTrees();
  // 1. 用户自定义注册（虚函数）
  registerNodesIntoFactory(p_->factory);
  // 2. 从目录加载 .so 插件
  RegisterPlugins(p_->params, p_->factory, node_);
  // 3. 从目录加载 .xml 行为树
  RegisterBehaviorTrees(p_->params, p_->factory, node_);
  p_->factory_initialized_ = true;
}
```

### 3.3 插件加载机制

[`RegisterPlugins()`](behaviortree_ros2/src/bt_utils.cpp:134) 的工作流程：

```
YAML plugins 列表
    │
    ▼
遍历每个 "package_name/subfolder"
    │
    ▼
GetDirectoryPath() → 解析为完整路径
    │
    ▼
遍历目录中所有 .so 文件
    │
    ▼
LoadPlugin() → 动态加载共享库
    │
    ├── 检查符号 BT_PLUGIN_SYMBOL ("BT_RegisterFromPlugin")
    │   └── 调用: func(factory)                         ← 标准 BT.CPP 插件
    │
    └── 检查符号 BT_RegisterRosNodeFromPlugin
        └── 调用: func(factory, ros_params)             ← ROS 插件（带节点参数）
```

关键代码（[`bt_utils.cpp`](behaviortree_ros2/src/bt_utils.cpp:101)）：

```cpp
void LoadPlugin(BT::BehaviorTreeFactory& factory,
                const std::filesystem::path& file_path,
                BT::RosNodeParams params)
{
  BT::SharedLibrary loader(file_path.string());
  if(loader.hasSymbol(BT::PLUGIN_SYMBOL)) {
    // 标准 BT.CPP 插件
    auto func = (Func)loader.getSymbol(BT::PLUGIN_SYMBOL);
    func(factory);
  }
  else if(loader.hasSymbol(BT::ROS_PLUGIN_SYMBOL)) {
    // ROS 插件，需要传入 RosNodeParams
    auto func = (Func)loader.getSymbol(BT::ROS_PLUGIN_SYMBOL);
    func(factory, params);
  }
}
```

### 3.4 XML 树加载机制

[`RegisterBehaviorTrees()`](behaviortree_ros2/src/bt_utils.cpp:162) 的工作流程：

```
YAML behavior_trees 列表
    │
    ▼
遍历每个 "package_name/subfolder"
    │
    ▼
GetDirectoryPath() → 解析为完整路径
    │
    ▼
LoadBehaviorTrees() → 遍历目录中所有 .xml 文件
    │
    ▼
factory.registerBehaviorTreeFromFile(path)
    │
    ▼
BT.CPP 解析 XML，提取 BehaviorTree ID，注册到工厂
```

关键代码（[`bt_utils.cpp`](behaviortree_ros2/src/bt_utils.cpp:79)）：

```cpp
void LoadBehaviorTrees(BT::BehaviorTreeFactory& factory,
                       const std::string& directory_path)
{
  for(const auto& entry : directory_iterator(directory_path)) {
    if(entry.path().extension() == ".xml") {
      factory.registerBehaviorTreeFromFile(entry.path().string());
      RCLCPP_INFO(kLogger, "Loaded BehaviorTree: %s",
                  entry.path().filename().c_str());
    }
  }
}
```

**重要**：XML 中 `<BehaviorTree ID="xxx">` 的 `ID` 就是后续 `factory.createTree("xxx")` 使用的树名。

### 3.5 运行时执行树

当 Action Client 发送 Goal 时，[`execute()`](behaviortree_ros2/src/tree_execution_server.cpp:176) 方法被调用：

```cpp
void TreeExecutionServer::execute(
    const std::shared_ptr<GoalHandleExecuteTree> goal_handle)
{
  const auto goal = goal_handle->get_goal();

  // 1. 如果参数有变，重新注册
  if(p_->param_listener->is_old(p_->params)) {
    executeRegistration();
  }

  // 2. 创建黑板（父级为全局黑板）
  auto root_blackboard = BT::Blackboard::create(p_->global_blackboard);

  // 3. 根据 target_tree 名称创建树实例
  p_->tree = p_->factory.createTree(goal->target_tree, root_blackboard);

  // 4. 调用用户钩子
  onTreeCreated(p_->tree);

  // 5. 启动 Groot2 调试发布器
  p_->groot_publisher = std::make_shared<BT::Groot2Publisher>(p_->tree, ...);

  // 6. 循环 tick
  while(rclcpp::ok() && status == BT::NodeStatus::RUNNING) {
    if(goal_handle->is_canceling()) {
      // 处理取消
      p_->tree.haltTree();
      goal_handle->canceled(action_result);
      return;
    }

    status = p_->tree.tickExactlyOnce();

    // 调用用户钩子
    if(auto res = onLoopAfterTick(status); res.has_value()) { ... }
    if(auto res = onLoopFeedback(); res.has_value()) { ... }

    // 按频率休眠
    p_->tree.sleep(...);
  }

  // 7. 返回结果
  goal_handle->succeed(action_result);  // 或 abort
}
```

## 4. 四种 ROS 节点封装基类

### 4.1 RosActionNode（Action 客户端）

**头文件**：[`bt_action_node.hpp`](behaviortree_ros2/include/behaviortree_ros2/bt_action_node.hpp)

**模板参数**：ROS2 Action 类型（如 `btcpp_ros2_interfaces::action::Sleep`）

**必须实现的回调**：

| 回调               | 签名                                                       | 说明                                                         |
| ------------------ | ---------------------------------------------------------- | ------------------------------------------------------------ |
| `setGoal`          | `bool setGoal(Goal& goal)`                                 | 设置 Action Goal。返回 `false` 则触发 `onFailure(INVALID_GOAL)` |
| `onResultReceived` | `NodeStatus onResultReceived(const WrappedResult& result)` | 收到结果时调用，返回 SUCCESS 或 FAILURE                      |

**可选回调**：

| 回调         | 签名                                                      | 说明                                        |
| ------------ | --------------------------------------------------------- | ------------------------------------------- |
| `onFeedback` | `NodeStatus onFeedback(const shared_ptr<const Feedback>)` | 收到反馈时调用，默认返回 RUNNING            |
| `onFailure`  | `NodeStatus onFailure(ActionNodeErrorCode error)`         | 出错时调用，默认返回 FAILURE                |
| `onHalt`     | `void onHalt()`                                           | 节点被暂停时调用，`cancelGoal()` 会自动调用 |

**错误码**（`ActionNodeErrorCode`）：

| 枚举值                    | 含义                   |
| ------------------------- | ---------------------- |
| `SERVER_UNREACHABLE`      | Action Server 不可达   |
| `SEND_GOAL_TIMEOUT`       | 发送 Goal 超时         |
| `GOAL_REJECTED_BY_SERVER` | Goal 被服务器拒绝      |
| `ACTION_ABORTED`          | Action 被中止          |
| `ACTION_CANCELLED`        | Action 被取消          |
| `INVALID_GOAL`            | `setGoal()` 返回 false |

**默认端口**：

```cpp
static PortsList providedBasicPorts(PortsList addition) {
  PortsList basic = { InputPort<std::string>("action_name", "", "Action server name") };
  basic.insert(addition.begin(), addition.end());
  return basic;
}
```

### 4.2 RosServiceNode（Service 客户端）

### 6.1 总体步骤


### 6.2 步骤一：设计 XML 决策树

在你的包中创建 `behavior_trees/` 目录，放入 XML 文件：


| `BTCPP_format="4"`      | 必须，声明 BT.CPP 4.x 格式             |

```cpp
#include "geometry_msgs/msg/pose_stamped.hpp"

public:
  NavigateToGoal(const std::string& name, const BT::NodeConfig& conf,
                 const BT::RosNodeParams& params)
    : RosActionNode<NavigateToPose>(name, conf, params)
  {}

  // 声明端口：继承 action_name 端口，再添加 goal 端口

      BT::InputPort<geometry_msgs::msg::PoseStamped>("goal", "Target pose")
    });
```

## 7. 关键设计模式与最佳实践

### 7.1 非阻塞异步设计

所有 ROS 通信节点（Action/Service）在 `tick()` 中都采用异步模式：

```
tick() 第一次调用 (IDLE → RUNNING)
    │
    ├── 构造 Goal/Request
    ├── async_send_goal() / async_send_request()
    └── 返回 RUNNING
    
tick() 后续调用 (RUNNING)
    │
    ├── callback_executor.spin_some()    ← 处理回调（非阻塞）
    ├── 检查 future 是否完成
    │   ├── 未完成 → 返回 RUNNING
    │   └── 已完成 → 调用 onResultReceived() → 返回 SUCCESS/FAILURE
    └── 检查超时 → 调用 onFailure()
```

**为什么非阻塞很重要**：BT 的 `tick()` 是顺序执行的，如果一个节点阻塞了，整个树的其他节点（包括 Timeout 装饰器）都无法执行。

### 7.2 客户端复用机制

框架使用全局注册表来避免重复创建 ROS 客户端：

```cpp
// 每个 RosActionNode/RosServiceNode 类内部
using ClientsRegistry = std::unordered_map<std::string, std::weak_ptr<ClientInstance>>;
static ClientsRegistry& getRegistry();
```

键名为 `node_fully_qualified_name + "/" + action_name`。如果同一个 Action Server 被多个 BT 节点使用，它们会共享同一个客户端实例。

### 7.3 端口名称解析策略

Action/Service 名称的确定遵循三级优先级：

```
1. XML 中硬编码的端口值
   <SleepAction action_name="sleep_service"/>
   
2. XML 中的黑板引用（运行时动态解析）
   <SleepAction action_name="{my_action_name}"/>
   
3. RosNodeParams.default_port_value
   （通过 YAML 配置或代码设置）
```

**判断逻辑**（以 `RosActionNode` 为例）：

```cpp
RosActionNode::RosActionNode(..., const RosNodeParams& params) {
  auto portIt = config().input_ports.find("action_name");
  if(portIt != ...) {
    const auto& bb_name = portIt->second;
    if(isBlackboardPointer(bb_name)) {
      action_name_should_be_checked_ = true;  // 延迟到 tick 时解析
    } else if(!bb_name.empty()) {
      createClient(bb_name);                   // 硬编码值，立即创建
    }
  }
  if(!client_instance_ && !params.default_port_value.empty()) {
    createClient(params.default_port_value);    // 使用默认值
  }
}
```

### 7.4 Blackboard 使用

行为树使用两级黑板结构：

```
全局黑板 (global_blackboard)          ← 持久存在，跨树执行
    │
    └── 根黑板 (root_blackboard)      ← 每次执行创建，树销毁时释放
        │
        └── 子树黑板 (subtree_bb)     ← SubTree 自动创建
```

在 `TreeExecutionServer` 中：

```cpp
// 全局黑板可在构造函数中通过订阅者更新
globalBlackboard()->set("battery_level", msg->data);

// 执行时，根黑板继承全局黑板
auto root_blackboard = BT::Blackboard::create(p_->global_blackboard);
p_->tree = p_->factory.createTree(goal->target_tree, root_blackboard);
```

### 7.5 Groot2 可视化调试

框架自动集成了 `BT::Groot2Publisher`，在树创建时启动：

```cpp
p_->groot_publisher = std::make_shared<BT::Groot2Publisher>(
    p_->tree, p_->params.groot2_port);
```

使用 [Groot2](https://www.behaviortree.dev/groot/) 工具连接到配置的端口（默认 `1667`）即可实时查看树执行状态。

## 8. 常见问题排查

### Q1：XML 中的节点名无法识别

**原因**：插件未正确加载

**检查清单**：

- [ ] `.cpp` 末尾是否有 `CreateRosNodePlugin(ClassName, "NodeName")` 或 `BT_REGISTER_ROS_NODES` 宏
- [ ] CMakeLists.txt 是否有 `target_compile_definitions(xxx PRIVATE BT_PLUGIN_EXPORT)`
- [ ] 插件安装路径是否正确：`share/${PROJECT_NAME}/bt_plugins`
- [ ] YAML 中 `plugins` 列表是否包含 `package_name/bt_plugins`
- [ ] XML 中节点名是否与注册名完全一致（大小写敏感）

### Q2：Action Server 不可达

**原因**：Action Server 未启动，或名称不匹配

**检查方法**：

```bash
# 查看可用的 Action Server
ros2 action list
# 检查 Action Server 类型
ros2 action info /sleep_service
```

**确认**：XML 中 `action_name` 值是否与实际 Action Server 名称一致。

### Q3：SubTree 找不到

**原因**：引用的树 XML 未加载

**检查**：

- [ ] 子树 XML 是否在 `behavior_trees` 目录中
- [ ] 子树 XML 中 `<BehaviorTree ID="xxx">` 的 ID 是否与 `<SubTree ID="xxx"/>` 匹配
- [ ] YAML 配置是否包含该目录

### Q4：节点超时

**调整参数**：

```yaml
ros_plugins_timeout: 5000  # 增大超时值 (ms)
```

或在代码中设置：

```cpp
RosNodeParams params;
params.server_timeout = std::chrono::milliseconds(5000);
params.wait_for_server_timeout = std::chrono::milliseconds(5000);
```

### Q5：动态更新树/插件

`TreeExecutionServer` 支持运行时重新加载。当 ROS 参数被修改时，下一次执行前会自动调用 `executeRegistration()` 重新加载：

```cpp
// tree_execution_server.cpp - execute()
if(p_->param_listener->is_old(p_->params)) {
  executeRegistration();  // 重新加载插件和树
}
```

可通过 `ros2 param set` 动态修改参数来触发重新加载。
