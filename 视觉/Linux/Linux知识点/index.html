    07. Linux 目录结构 📂

    一. 结构特点

    1. linux 的文件系统是采用级层式的树状目录结构，在此结构中的最上层是根目录 /，然后在此目录下再创建其他的目录（全是目录）。
    2. 在 linux 的世界中，一切皆文件。对主体的文件夹需要背下来。

    09. vi 和 vim 编辑器 📝

    Linux 系统会内置 vi 文本编辑器，vim 是 vi 的增强版。

    一. vi 和 vim 常用的三种模式

    1. 正常模式：
    2. 以 vim 打开一个档案就直接进入一般模式了（这是默认的模式），在这个模式中，你可以使用【上下左右】按键来移动光标，你可以使用 [删除字符] 或【删除整行】来处理档案内容，也可以使用【复制、粘贴】来处理你的文件数据。
    3. 插入模式：
    4. 按下 i, I, o, O, a, A, r, R 等任何一个字母之后才会进入编辑模式，一般来说按 i 即可。
    5. 命令行模式：
    6. 在这个模式当中，可以提供你相关指令，完成读取、存盘、替换、离开 vim、显示行号等的动作。

    操作流程：

    • 在终端中输入 vim 语句进入正常模式。
    • 输入 i，进入插入模式。
    • 内容编辑完成之后，点击 Esc，进入命令行模式，然后输入 :wq（w 代表 write，q 代表退出）。

    10. 开机、重启和用户登录注销 🔌

    一. 关机和重启命令

    1. 基本介绍：
    2. shutdown -h now：立刻进行关机（h 是 halt 的缩写）。
    3. shutdown -h 1：1分钟后会关机 (h 应该是 halt，原文写 r 是 reboot 的缩写，此处修正为 h 对应关机)。
    4. shutdown -r now：现在重新启动计算机。
    5. halt：关机。
    6. reboot：现在重新启动计算机。
    7. sync：把内存数据同步到磁盘。
    8. 注意细节：
    9. 不管是重启下系统还是关闭系统，首先要运行 sync 命令，把内存中的数据写到磁盘中。
    10. 如果直接输入 shutdown，默认 shutdown -h 1。

    二. 用户登录和注销

    1. 基本介绍：
    2. 登录时尽量少用 root 账号登录，因为它是系统管理员，最大的权限，避免操作失误，可以利用普通用户登录，“登录后再用 su - 用户名 命令来切换成系统管理员身份”。
    3. 在提示符下输入 logout 即可注销用户。
    4. 使用细节：
    5. logout 注销指令在图形运行级别无效（就是直接鼠标右键点击终端打开的界面），在运行级别 3 下有效。
    6. 如果登录非管理员用户，然后再切换到管理员用户，此时使用 logout，只会从管理员用户退出，然后回到非管理员用户中，如果在非管理员用户中使用 logout，会直接退出登录。

    11. 用户管理 👤

    Linux 是一个多用户多任务的操作系统，任何一个要使用系统资源的用户，都必须首先像系统管理员申请一个账号，然后以这个账号的身份进入系统。

    1. 添加用户：
    2. 基本语法：useradd 用户名。
    3. 当创建用户成功后，会自动的创建和用户同名的家目录（默认家目录在 /home/新建的目录名）。
    4. 也可以通过 useradd -d 指定目录 新建的用户名，给新创建的用户指定家目录。
    5. 指定/修改密码：
    6. 基本语法：passwd 用户名。
    7. 注意：普通用户无法登录，需要设密码。
    8. pwd 是用于显示当前你在哪个目录下。
    9. 删除用户：
    10. userdel 用户名：用于删除用户，但保留用户家目录。
    11. userdel -r 用户名：用于删除用户以及用户家目录（一般是建议保留家目录）。
    12. 查询用户信息指令：
    13. 基本语法：id 用户名。
    14. 切换用户：
    15. su - 切换用户（如果当前用户权限不够，先切换到最高权限目录，再切换到对应用户）。
    16. 从权限高的用户切换到权限低的用户，不需要输入密码，反之需要。
    17. 退出命令为 exit / logout。
    18. 查看当前用户/登录用户：
    19. 基本语法：whoami（只会显示谁登录了）/ who am I（会显示登录时间和谁）。
    20. 显示的是你第一次登录到这个用户的信息（屏蔽掉切换的用户）。
    21. 用户组：
    22. 类似于角色，系统可以对有共性/权限的多个用户进行统一的管理。
    23. 新增组：groupadd 组名。
      • 如果创建用户时没有其他指令，会直接创建一个以用户名来命名的组。
    24. 删除组：groupdel 组名。
    25. 增加用户时直接加上组：useradd -g 用户组 用户名。
    26. 修改用户的组：usermod -g 用户组 用户名。
    27. 用户和组相关的文件：
    28. /etc/passwd 文件：用户 (user) 的配置文件，记录用户的各种信息。
      • 每行的含义：用户名：口令：用户标识号（id）：注释性描述：主目录：登录Shell。
    29. /etc/shadow 文件：口令配置文件。
      • 每行的含义：登录名：加密口令：最后一次修改时间：最小时间间隔：最大时间间隔：警告时间：不活动时间：失效时间：标志。
    30. /etc/group 文件：组的配置文件，记录 Linux 包含的组的信息。
      • 每行含义：组名：口令：组标识号：组内用户列表。
    31. 可以使用 vim 来查看文件内容。加密口令其实就是加密用户的密码。

    12. 找回 root 密码 🔑

    1. 启动系统，进入开机界面，在界面中按 e 进入编辑界面。
    2. 使用键盘上的上下键光标往下移动，找到以 “Linux16” 开头内容所在的行数。
    3. 在行的最后面输入 :init=/bin/sh。
    4. 输入完成后，直接按快捷键：ctrl+x 进入单用户模式。
    5. 在鼠标闪烁的位置输入：mount -o remount,rw /，完成后按回车键。
    6. 在新的一行最后面输入：passwd root，完成后按回车键，输入密码，再次确认密码。
    7. 密码修改成功后，会显示 passwd... 的样式。
    8. 继续在光标闪烁的位置上输入 touch /.autorelabel。
    9. 输入 exec /sbin/init，完成后按回车键。
    10. 等待系统自动修改密码，完成后，系统会自动重启，新的密码生效。

    13. Linux 帮助指令 ❓

    1. man 获得帮助信息：
    2. 基本语法：man [命令或配置文件]（功能描述：获得帮助信息）。
    3. 示例：man ls。
    4. help 指令：
    5. 基本语法：help 指令（功能描述：获得 shell 内置命令的帮助信息）。
    6. 如果没有全部显示出来可以按空格键，使其继续显示。

    14. 文件目录类 📁

    1. pwd 指令：显示当前工作目录的绝对路径。
    2. ls 指令：ls [选项][目录或是文件]。
    3. -a: 显示当前目录所有的文件和目录，包括隐藏的。
    4. -l: 以列表的方式显示信息。
    5. cd 指令：切换到指定目录。
    6. cd ~ 或者 cd：回到自己的家目录。
    7. cd ..：回到当前目录的上一级目录（可以这样写 cd ../../..）。
    8. mkdir 指令：用于创建目录。
    9. -p: 创建多级目录。
    10. 示例：mkdir -p /home/animal/tiger。
    11. rmdir 指令：删除空目录。
    12. 注意：rmdir 删除的是空目录，如果目录下有内容时是无法删除的。
    13. 提示：如果需要删除非空目录，需要使用 rm -rf 要删除的目录。
    14. touch 指令：创建空文件。
    15. cp 指令：拷贝文件到指定目录。
    16. 语法：cp [选项] source dest。
    17. -r：递归复制整个文件夹。
    18. 强制覆盖不提示的方法：\cp。
    19. rm 指令：移除文件或目录。
    20. -r: 递归删除整个文件夹。
    21. -f: 强制删除不提示。
    22. mv 指令：移动文件与目录或重命名。
    23. mv oldName newName (重命名)。
    24. mv /temp/movefile /targetFolder (移动文件)。
    25. cat 指令：查看文件内容。
      • -n: 显示行号。
      • 细节：cat 只能浏览文件，一般会带上管道命令 | more（enter 多显示一行，空格键是换页）。
    26. more 指令：基于 Vi 编辑器的文本过滤器，按页显示。
    27. less 指令：分屏查看，比 more 强大。不是一次加载整个文件，适合大型文件。
    28. echo 指令：输出内容到控制台。
    29. head 指令：显示文件的开头部分（默认前 10 行）。
      • head -n 5 文件（查看前 5 行）。
    30. tail 指令：输出文件中尾部的内容（默认后 10 行）。
      • tail -n 5 文件（查看后 5 行）。
      • tail -f 文件（实时追踪该文档的所有更新）。
    31. > 和 >> 指令：
      • > 输出重定向（覆盖写）。示例：ls -l > a.txt。
      • >> 追加。示例：ls -al >> aa.txt。
    32. ln 指令：软链接，类似于 windows 里的快捷方式。
      • ln -s [原文件或目录] [软链接名]。
      • 删除软链接：rm myroot。
    33. history 指令：查看已执行过的历史命令。
      • history 10：查看最近 10 个。
      • !5：将序号为 5 的命令重新执行一遍。

    15. 时间日期类 📅

    1. date 指令 - 显示当前日期：
    2. date（显示当前时间）。
    3. date +%Y（显示当前年份）。
    4. date +%m（显示当前月份）。
    5. date +%d（显示当前是哪一天）。
    6. date "+%Y-%m-%d %H:%M:%S"（显示年月日时分秒）。
    7. date 指令 - 设置日期：
    8. date -s 字符串时间。
    9. cal 指令：查看日历。
    10. cal 2020（显示 2020 年的日历）。

    16. 搜索查找类 🔍

    1. find 指令：从指定目录向下递归地遍历。
    2. -name <查询文件名>：按照指定的文件名查找。
    3. -user <用户名>：查找属于指定用户名所有文件。
    4. -size <文件大小>：按大小查找（+n 大于，-n 小于，n 等于，单位 k, M, G）。
    5. locate 指令：快速定位文件路径，基于数据库。
    6. 特别说明：第一次运行前，必须使用 updatedb 指令创建 locate 数据库。
    7. which 指令：查看某个指令在哪个目录下。
    8. grep 指令和管道符号 |：
    9. grep 过滤查找，管道符 | 表示将前一个命令的处理结果传递给后面的命令处理。
    10. -n: 显示匹配行及行号。
    11. -i: 忽略字母大小写。
    12. 示例：cat 路径 | grep "yes" 或 grep -n "yes" 文件路径。

    17. 压缩和解压类 📦

    1. gzip / gunzip 指令：
    2. gzip 文件路径：压缩文件，只能将文件压缩为 *.gz 文件。
    3. gunzip 文件.gz：解压缩文件。
    4. zip / unzip 指令：
    5. zip [选项] xxx.zip 将要压缩的内容。
      • -r: 递归压缩，即压缩目录。
    6. unzip [选项] xxx.zip。
      • -d <目录>: 指定解压后文件的存放目录。
    7. tar 指令：打包指令，最后打包的文件是 .tar.gz。
    8. -c: 产生 .tar 打包文件。
    9. -v: 显示详细信息。
    10. -f: 指定压缩后的文件名。
    11. -z: 打包同时压缩。
    12. -x: 解包 .tar 文件。
    13. 示例（压缩）：tar -zcvf xxx.tar.gz 文件路径。
    14. 示例（解压）：tar -zxvf /home/myhome.tar.gz -C /opt/tmp2。

    18. 组管理和权限管理 🔐

    1. 基本概念：每个用户必须属于一个组。每个文件有所有者、所在组、其他组的概念。
    2. 所有者：
    3. 查看指令：ls -ahl（第一个是所有者，第二个是所在组）。
    4. 修改所有者：chown 用户名 文件名。
    5. 所在组：
    6. 修改所在组：chgrp 组名 文件名。
    7. 其他组：除所有者和所在组外的用户。
    8. 权限介绍 (-rwxrm-r--)：
    9. 第 0 位：文件类型（l 链接, d 目录, c 字符设备, b 块设备, - 普通文件）。
    10. 第 1-3 位 (User)：所有者权限。
    11. 第 4-6 位 (Group)：所属组权限。
    12. 第 7-9 位 (Other)：其他用户权限。
    13. rwx 含义：
      • r (4): 读。
      • w (2): 写（对目录来说是创建/删除/重命名）。
      • x (1): 执行（对目录来说是进入）。
    14. 修改权限 - chmod：
    15. 方式一：chmod u=rwx,g=rx,o=x 文件名。
    16. 方式二：chmod 751 文件名 (4+2+1=7)。
    17. 修改所有者/所在组：
    18. chown newowner:newgroup 文件。
    19. -R：递归生效。

    19. 权限管理应用实例 👮

    • 场景：police, bandit 组；jack, jerry (警察)；xh, xq (土匪)。
    • 流程：
    • 创建组。
    • 创建用户。
    • jack 创建文件，自己 r+w，本组 r，其他组无权限。
    • 修改权限测试读写。

    21. 任务调度机制 (at) ⏲️

    2. at 是一次性定时计划任务，由 atd 守护进程执行。

    27. 进程管理 📈

    一. 基本介绍

    • 每个进程都有一个 ID 号 (pid)。
    • 进程分为前台和后台。

    二. 显示系统执行的进程 (ps)

    • ps -aux | grep xxx: 查看进程。
    • ps -ef: 全格式显示，可以查看父进程。
    • 参数详解：
    • USER: 用户。
    • PID: 进程号。
    • %CPU / %MEM: 占用率。
    • STAT: 状态 (S-睡眠, R-运行, Z-僵死)。

    三. 终止进程

    • kill [选项] 进程号。
    • -9: 强迫立即停止。
    • killall 进程名称 (支持通配符)。

    四. 查看进程树

    • pstree [选项]。
    • -p: 显示 PID。
    • -u: 显示所属用户。

    28. 服务管理 ⚙️

    一. 介绍

    • 服务本质是运行在后台的进程，监听端口，称为守护进程。
    • service 指令 (CentOS 7.0 之前常用):
    • service 服务名 [start|stop|restart|reload|status]。
    • 位置：/etc/init.d。

    二. 服务的运行级别

    • 常用级别：3 (多用户命令行) 和 5 (图形界面)。
    • 0: 停机。
    • 6: 重启。
    • CentOS 7 简化为 multi-user.target (3) 和 graphical.target (5)。

    三. systemctl 管理指令 (CentOS 7+)

    • 语法：systemctl [start|stop|restart|status] 服务名。
    • 设置自启动：
    • systemctl list-unit-files: 查看状态。
    • systemctl enable 服务名: 设置开机启动。
    • systemctl disable 服务名: 关闭开机启动。

    四. 打开/关闭指定端口 (Firewall)

    • 打开端口：firewall-cmd --permanent --add-port=端口号/协议。
    • 关闭端口：firewall-cmd --permanent --remove-port=端口号/协议。
    • 重新载入生效：firewall-cmd --reload。
    • 查询端口：firewall-cmd --query-port=端口/协议。
